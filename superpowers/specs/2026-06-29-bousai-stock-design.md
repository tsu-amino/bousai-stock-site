# 防災ストック管理アプリ 実装仕様書

最終更新: 2026-06-29  
ステータス: UI/UX確定、実装フェーズへ引き継ぎ可能

---

## 1. 概要

防災用品(非常食・飲料水・電池・医薬品・防災グッズ等)の期限・点検時期を一元管理するスマートフォンアプリ。

| 項目 | 内容 |
|---|---|
| 目的 | 防災用品の期限切れ・点検漏れを防止する |
| 対象範囲 | 非常食、飲料水、電池・電源、医薬品・救急、防災用品(点検が必要なもの: ヘルメット等) |
| 動作環境 | スマートフォン(iOS / Android)、完全オフライン動作 |
| サーバー | 不要。すべてのデータは端末内のみに保存 |
| 通知 | 期限が近づいたらプッシュ通知(ローカル通知) |

---

## 2. 技術スタック

| 項目 | 採用技術 | 補足 |
|---|---|---|
| フレームワーク | Expo (React Native) | ストア審査用のネイティブビルドはEAS Buildで対応。OTA更新も活用可能 |
| 言語 | TypeScript | KrayNote既存プロジェクトと統一 |
| ローカルDB | expo-sqlite | 構造化データ、検索・ソート・フィルタに対応 |
| 状態管理 | Zustand | 軽量、SQLiteとの併用がしやすい |
| 画像 | expo-image-picker + expo-file-system | 撮影/ライブラリ選択した画像を端末内ディレクトリにコピーして永続化 |
| 通知 | expo-notifications | ローカル通知のみ。サーバー(プッシュサービス)経由は不要 |
| ナビゲーション | expo-router | ファイルベースルーティング |
| 外部API | 使用しない | 将来の拡張に備え services/ 層で抽象化 |

### コード規約の方針

- ディレクトリ構成、ESLint/Prettier設定、命名規則はKrayNoteの他プロジェクト(VidFlow等)に準拠
- 外部API連携は行わない。将来的な拡張(写真からのアイテム名自動推測等)に備え `services/` 層でAPI呼び出しを抽象化し、差し替え可能な設計にする

## 3. データ構造(SQLiteスキーマ)

### 3.1 categories(カテゴリ)

固定マスタ。アプリ初回起動時にシードする。

| カラム | 型 | 説明 |
|---|---|---|
| id | TEXT PK | `food` / `water` / `battery` / `medical` / `gear` |
| name | TEXT | 表示名 |
| icon | TEXT | Tabler iconクラス名 |
| color | TEXT | テーマカラーキー |
| sort_order | INTEGER | 表示順 |

### 3.2 storage_locations(保管場所マスタ)

定型選択用。ユーザーが追加・編集可能。

| カラム | 型 | 説明 |
|---|---|---|
| id | TEXT PK | UUID |
| name | TEXT | 例: 玄関収納、物置、寝室、車のトランク |
| is_default | INTEGER | 初期シードかどうか(0/1) |
| sort_order | INTEGER | 表示順 |

初期シード例: 玄関収納、リビング、寝室、キッチン、物置、車のトランク、その他

### 3.3 items(アイテム)

| カラム | 型 | 説明 |
|---|---|---|
| id | TEXT PK | UUID |
| category_id | TEXT FK | categories.id |
| name | TEXT | アイテム名(例: 2Lミネラルウォーター) |
| quantity | INTEGER | 数量 |
| unit | TEXT NULL | 単位(本、個、袋など。任意入力) |
| expiry_date | TEXT | 期限日(ISO 8601: YYYY-MM-DD) |
| expiry_type | TEXT | `expiry`(消費/使用期限) または `inspection`(点検時期) |
| storage_location_id | TEXT FK NULL | storage_locations.id |
| storage_note | TEXT NULL | 保管場所の補足テキスト(例: 下段右側のボックス) |
| photo_path | TEXT NULL | 端末内の画像ファイルパス(1アイテム1枚) |
| memo | TEXT NULL | 自由メモ |
| notify_days_before | INTEGER | 通知タイミング(期限の何日前。デフォルト14) |
| notification_id | TEXT NULL | expo-notificationsが発行する通知ID |
| created_at | TEXT | 作成日時 |
| updated_at | TEXT | 更新日時 |

### 3.4 インデックス

```sql
CREATE INDEX idx_items_category ON items(category_id);
CREATE INDEX idx_items_expiry ON items(expiry_date);
```

## 4. 画面構成

```
app/
├── (tabs)/
│   ├── index.tsx          # ホーム画面(カテゴリ別サマリー)
│   ├── list.tsx           # 全アイテム一覧(フィルタ・検索)
│   ├── notifications.tsx  # 通知設定・通知済み履歴
│   └── settings.tsx       # 保管場所マスタ管理、アプリ設定
├── item/
│   ├── [id].tsx            # アイテム詳細
│   ├── [id]/edit.tsx       # アイテム編集
│   └── new.tsx             # 新規アイテム追加
└── category/
    └── [id].tsx            # カテゴリ別アイテム一覧
```

### 4.1 ホーム画面(index.tsx)

- 上部サマリー: 「期限切れ」件数 / 「30日以内」件数(2カラムのメトリクスカード)
- 警告バナー: 直近で最も緊急度の高いアイテムを1件ピックアップして表示
- カテゴリ一覧: 5カテゴリをカードで表示。各カードに件数・直近期限・残り日数バッジ
- バッジの色ルール: 期限切れ=赤、30日以内=オレンジ、それ以外=グレー
- カテゴリカードタップ → category/[id].tsx
- FAB相当の「アイテムを追加」ボタン → item/new.tsx

### 4.2 カテゴリ別一覧(category/[id].tsx)

- 該当カテゴリのアイテムをカード形式で一覧表示
- 各カードに角丸サムネイル(写真なければカテゴリアイコン代替)、アイテム名、保管場所、期限バッジ
- 並び順: 期限が近い順(デフォルト)。ソート切替オプション

### 4.3 アイテム詳細(item/[id].tsx)

- 写真エリア(上部、大きめ表示。未登録時はプレースホルダー)
- アイテム名・カテゴリ
- 期限切れ: 赤背景の警告ボックス / 30日以内: オレンジ系警告ボックス
- 詳細情報: 保管場所(定型名+補足)、数量、メモ
- 通知設定表示(「期限の○日前に通知」)
- 「補充・期限を更新する」ボタン → 期限更新用の簡易モーダル
- 編集アイコン(ヘッダー) → item/[id]/edit.tsx

### 4.4 新規追加・編集(item/new.tsx, item/[id]/edit.tsx)

- カテゴリ選択(必須)
- アイテム名(必須)
- 数量・単位(任意)
- 期限日(必須、日付ピッカー)。「防災用品・点検」カテゴリ時はラベルを「次回点検日」に切替
- 保管場所: 定型選択 + 補足テキスト(この画面から新規追加も可能)
- 写真: 撮影またはライブラリから選択(1枚のみ)
- メモ(任意)
- 通知タイミング: 3日/7日/14日/30日から選択(デフォルト14日)
- 保存時: DB更新 → 通知再スケジュール → 旧写真削除・新写真コピー

### 4.5 通知画面(notifications.tsx)

- 通知済み・通知予定アイテムのリスト(期限が近い順)
- グローバル通知の有効/無効切替

### 4.6 設定画面(settings.tsx)

- 保管場所マスタの追加・編集・削除
- アプリについて

## 5. 通知ロジック

- expo-notificationsのローカル通知をアイテムごとに1件スケジュール
- 発火条件: `expiry_date - notify_days_before日` の朝9時(固定)
- 通知本文: 「{アイテム名}の期限が近づいています({期限日})」
- 期限切れ後は再通知しない(初回の事前通知のみ)
- アプリ起動時に全アイテムをスキャンし、通知とDBの整合性チェック(端末再起動対応)

## 6. デザイン仕様

### 6.1 カラー方針

防災を意識させるオリーブグリーン基調。緊急度を示す赤・オレンジは警告専用に予約。

| 用途 | 色 |
|---|---|
| ヘッダーアイコン・アクティブタブ | オリーブグリーン系(濃) |
| 期限切れ | 赤(背景は薄赤、文字は濃赤) |
| 30日以内(警告) | オレンジ/アンバー(背景は薄橙、文字は濃橙) |
| それ以外 | グレー(ニュートラル) |
| カテゴリアイコン背景 | カテゴリごとに異なる薄色(非常食=コーラル系、飲料水=ブルー系、電池=アンバー系、防災用品=グリーン系、医薬品=ピンク系) |

### 6.2 タイポグラフィ・レイアウト

- 角丸: カードは12px、バッジ等の小要素はpill形状
- カード: 白背景、薄いボーダー、十分な余白
- フォントウェイトは2種類のみ(regular / medium)
- 一覧は軽量、詳細は充実の情報設計
